Create `lib/client-logger.ts` - a client-safe error logging utility using Sentry SDK (Pino is server-only).

Implementation:
1. Import Sentry SDK: `import * as Sentry from "@sentry/nextjs";`
2. Export `logClientError(error: unknown, context: { component?: string; action?: string; userId?: string; [key: string]: unknown })` function
3. Behavior:
   - In production: Call `Sentry.captureException(error, { tags: { component, action }, extra: context })`
   - In development: Call `console.error()` with formatted output for visibility
4. Add JSDoc explaining usage pattern and when to use vs server-side logger
5. NO "server-only" directive (this is client-safe by design)

Pattern from research (28-RESEARCH.md Pattern 4):
```typescript
export function logClientError(
  error: unknown,
  context: {
    component?: string;
    action?: string;
    userId?: string;
    [key: string]: unknown;
  }
) {
  if (process.env.NODE_ENV === "production") {
    Sentry.captureException(error, {
      tags: {
        component: context.component,
        action: context.action,
      },
      extra: context,
    });
  }

  if (process.env.NODE_ENV === "development") {
    console.error(`[${context.component}] ${context.action}:`, error, context);
  }
}
```

Why client-side utility needed: Components and hooks run in browser where Pino (Node.js-only) cannot be imported. Sentry SDK is the client-safe structured logging mechanism.
