  <name>Task 1: Create use-voice-call hook with real-time voice loop</name>
  <files>hooks/use-voice-call.ts</files>
  <action>
Create a custom hook based on the pattern in the research document (32-RESEARCH.md lines 369-536). This hook manages the voice call lifecycle:

1. **State machine:** idle → listening → thinking → speaking → (loop back to listening)
2. **SpeechRecognition integration:**
   - Detect browser support (window.SpeechRecognition || window.webkitSpeechRecognition)
   - Configure: continuous=true, interimResults=true, lang="en-US"
   - Handle onresult: collect final transcript, send to AI
   - Handle onerror: log, auto-retry on transient errors (no-speech, aborted)
   - Handle onend: auto-restart if still in listening state
3. **AI response flow:**
   - POST to /api/realtime/stream with { message: text, botType: executive }
   - Use csrfFetch from useCsrf hook
   - Parse response: { audioUrl, audioData }
   - Play audio via Audio() API, transition to "speaking" state
   - On audio.onended, return to "listening" state
4. **Cleanup:**
   - stopCall() stops recognition, pauses audio, resets state
   - startCall() requests mic permission, then starts listening
   - useEffect cleanup calls stopCall() on unmount

Export: `useVoiceCall({ executive: BotType })` returning `{ callState, transcript, startCall, stopCall, isListening, isThinking, isSpeaking }`

TypeScript types: CallState = "idle" | "listening" | "thinking" | "speaking"

Use logClientError from lib/client-logger.ts for error logging.
